## 2024-05-11 - [CRITICAL] Fix authorization bypass in API (Local Admin Endpoints Exposure)
**Vulnerability:** The `/api/admin/` endpoints in the Flask Student Portal were missing authorization or explicit IP-based access controls. Because Waitress listens on `0.0.0.0` to allow student access, these administrative endpoints were exposed to the entire network without authentication, allowing anyone to manage requests, notices, or bulk password resets.
**Learning:** Endpoints intended solely for local desktop application communication must explicitly enforce IP whitelisting when hosted within a web server that binds to public interfaces (`0.0.0.0`). Additionally, `X-Forwarded-For` headers must not be trusted for IP restriction checks unless operating securely behind a trusted reverse proxy, as they are easily spoofed by malicious clients.
**Prevention:** Always implement an `@app.before_request` hook or specific decorators to strictly validate `request.remote_addr` against `['127.0.0.1', '::1']` for local-only administrative APIs, and ensure CSRF bypass logic for these endpoints does not inadvertently open them to external abuse.
