## 2024-05-24 - [CRITICAL] Admin Endpoints Missing Authentication/Authorization
**Vulnerability:** The `/api/admin/*` endpoints in `LibraryApp/Web-Extension/student_portal.py` lacked explicit authentication or authorization. They were intended exclusively for the desktop application to access via `127.0.0.1`, but waitressed bound them to `0.0.0.0` or a public cloud routing environment could expose them, allowing unauthorized users to approve/reject requests, delete notices, and reset passwords.
**Learning:** Development-only or internal-app-only endpoints must explicitly verify the local origin or context in code. You cannot assume network isolation (like the desktop app's environment) will protect the web server, especially if the web server code can be deployed to a cloud environment (e.g. Render) or bound to all interfaces.
**Prevention:** Implement explicit IP checks or environment checks using `@app.before_request` for endpoints that are internal-only. A dedicated `enforce_admin_local_only` hook that checks for `127.0.0.1`, `::1` or server flags (`RENDER`, `IS_SERVER`) prevents these endpoints from being accessed externally.
