## 2024-05-24 - [IP Spoofing via X-Forwarded-For Header]
**Vulnerability:** The application was vulnerable to IP spoofing where an attacker could bypass rate limits by supplying fake IP addresses in the `X-Forwarded-For` header. The application was directly using the raw header string, or worse, manually extracting the *first* IP in the comma-separated list, which explicitly trusts the attacker-controlled spoofed IP.
**Learning:** `X-Forwarded-For` lists IPs in the order they were added. The first IP is the one supplied by the original client, making it trivial to spoof. Trusted reverse proxies append the real connection IP to the end of the list. Manual parsing is error-prone and often leads to extracting the attacker-controlled left-most IP.
**Prevention:** Never manually parse `X-Forwarded-For`. Always use secure, framework-provided middleware like Werkzeug's `ProxyFix` which correctly handles extracting the true client IP from the right side of the list based on the configured number of trusted proxies. This automatically and securely populates `request.remote_addr`.
